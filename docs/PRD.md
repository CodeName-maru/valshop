## 1. 배경

발로란트의 개인 상점은 매일 00:00(KST) 에 4개 스킨으로 로테이션되고, 주기적으로 야시장이 열린다. 현재는 이를 확인하려면 PC 를 켜고 발로란트 클라이언트에 로그인해야만 한다. 외출 중이거나 침대에 누워있을 때 "오늘 뭐 떴나" 를 보려면 상당한 마찰이 존재한다.

과거 `valstore.co`, `blitz.gg` 등의 서드파티가 이 역할을 했으나 라이엇의 비공식 엔드포인트 정책 변경으로 기능이 축소·중단되는 일이 반복됐다. 특히 **원하는 스킨이 떴을 때 푸시 알림을 받는** 기능을 제공하는 서비스는 드물어, 유저는 여전히 매일 수동으로 앱/사이트를 열어 확인해야 한다.

본 프로젝트는 **개인 사용 + 포트폴리오** 목적으로, 모바일에서 (a) 오늘의 상점을 즉시 확인하고 (b) 위시리스트 스킨이 뜨면 푸시 알림을 받는 경험을 만든다. 데드라인은 2026-04-26 (일) 이다.

## 2. 목표 사용자

- Primary: **스킨 과금에 적극적인 한국 (KR) 리전 발로란트 플레이어**. 본인 (개발자 자신) 포함. 사용 상황은 외출 중·침대에서 모바일로 상점을 가볍게 확인. 체감 사용 빈도는 **앱 직접 오픈은 이따금, 주된 상호작용은 푸시 알림 수신**.
- Secondary: 없음. 배포는 덤이며 타인 유입은 기대하지 않음.
- Not-target: TBD (§ 9 참조).

## 3. 목표 & Non-goals

- 목표: PC 를 켜지 않고 모바일에서 오늘 상점과 야시장을 **빠르고 가볍게** 확인. 원하는 스킨이 떴을 때 알림으로 능동 전달. 라이엇 계정 자격증명을 **서버에 원본 저장하지 않는** 신뢰 가능한 구조.
- 측정 가능 목표:
  - **TTI ≤ 3초**: 앱 오픈 후 상점 4개가 렌더되기까지
  - **위시리스트 푸시 ≤ 1시간**: 상점 로테이션 후 매칭 스킨 알림 도달까지 (Vercel Hobby Cron 무료 티어 제약 반영)
  - **월 호스팅 비용 $0**: Vercel / Supabase 무료 티어 내 운영
- Non-goals (이번 버전에서 명시적으로 **안** 함):
  - 야시장 이미지 공유 카드 생성
  - 가격 히스토리 / 출현 주기 통계
  - 친구 상점 공유 보기
  - 다국어 지원 (한국어만)
  - iOS 네이티브 앱 (PWA 로 한정)
  - 결제 / VP 충전 중개
  - 라이엇 계정 관리 (PW 변경 등 — 라이엇 공식 사이트로 링크)
  - VP 잔액 표시
  - 매치 기록 / 전적
  - 야시장 히스토리 (과거 출현 스킨 목록)

## 4. 범위

In-scope 는 2 트랙으로 분리한다.

- **MVP (주말 내 필수, ~2026-04-26)**:
  - Riot 비공식 auth flow 로그인 → PUUID 및 entitlements 토큰 획득
  - 토큰 암호화 저장으로 세션 유지 (재방문 시 자동 로그인)
  - 오늘의 상점 4개 카드 렌더 (이름, 가격 VP, 등급 티어 아이콘, 이미지)
  - 상점 갱신 카운트다운 타이머
  - `valorant-api.com` 메타데이터 매칭
  - Vercel 배포 + PWA 설치 가능
  - 로그아웃 버튼
  - 푸터에 "VAL-Shop 은 라이엇 게임즈와 무관한 팬메이드 프로젝트" 고지

- **Phase 2 (주말 이후 여력 시)**:
  - 스킨 검색 + 위시리스트 CRUD (Supabase 저장)
  - 주기 워커: 유저 상점 폴링 → 위시리스트 매칭 시 Web Push
  - 스킨 상세 뷰 (크로마, 고화질 이미지; 영상 링크 옵션)
  - 야시장 전용 뷰

Out-of-scope (영원히 안 함 또는 명시적 제외):
  - Non-goals 에 나열된 전 항목
  - 다른 리전 (NA/EU 등) 지원
  - 라이엇 계정 자격증명 (ID/PW) 서버 저장

시스템 경계: 외부 의존은 **Riot auth/store endpoints**, **valorant-api.com** (스킨 메타), **Supabase** (Phase 2 위시리스트), **Web Push 서비스** (Phase 2) 네 곳으로 한정.

## 5. 기능 요구사항

**MVP**:
- FR-1: 유저는 Riot 비공식 auth flow 를 통해 로그인할 수 있다. 비밀번호는 어느 서버에도 저장되지 않으며, 브라우저-Riot 간 직접 인증 후 토큰만 앱이 취득한다.
- FR-2: 유저는 최초 로그인 후 세션이 만료되기 전까지 재방문 시 자동 로그인된다. 토큰은 클라이언트 측 암호화 저장 (localStorage AES 또는 httpOnly cookie) + Phase 2 대비 서버측 Supabase vault (암호화) 에 보관한다.
- FR-3: 유저는 대시보드에서 오늘의 상점 4개 카드를 본다. 각 카드는 스킨 이름, 가격 (VP), 등급 티어 아이콘, 이미지를 포함한다.
- FR-4: 유저는 대시보드에서 상점 갱신까지 남은 시간을 초 단위 카운트다운으로 본다. ±1초 정확도.
- FR-5: 유저는 로그아웃 버튼으로 세션을 종료할 수 있다. 로그아웃 시 로컬·서버 저장 토큰이 즉시 파기된다.
- FR-6 (에러 처리): 토큰 만료 감지 시 자동으로 재로그인 플로우를 띄운다. Riot API 가 5xx / 429 를 반환하면 에러 화면을 보여주며 재시도 옵션을 제공한다. 로그인 실패 (2FA, 잘못된 자격증명 등) 시 에러 메시지와 재시도 UI 를 노출한다.

**Phase 2**:
- FR-7: 유저는 전체 스킨 카탈로그에서 스킨을 검색하고, 원하는 스킨을 위시리스트에 찜/해제할 수 있다. 위시리스트는 내 계정에만 연결되며 공유·관리자 기능은 없다.
- FR-8: 시스템은 주기적으로 각 유저의 상점을 폴링하여, 위시리스트 스킨이 감지되면 해당 유저에게 Web Push 알림을 1시간 이내 전달한다.
- FR-9: 유저는 스킨 카드를 탭하여 고화질 이미지와 크로마 색상 옵션을 확인할 수 있다. 인게임 영상 링크는 옵션.

## 7. 제약

- **시간**: 2026-04-26 (일) 까지 MVP 동작.
- **예산**: $0 (무료 티어만).
- **스택 고정**: Next.js (React, App Router), Vercel Serverless Functions, Supabase (Phase 2), `valorant-api.com` 메타데이터 API, Riot 비공식 auth endpoints.
- **인력**: 1인 (maru).
- **리전**: 한국 (KR) 한정. 리전 셀렉터 UI 없음.
- **C-RSO**: Riot 공식 OAuth2 RSO 는 승인된 파트너만 접근 가능하여 사용 불가. 본 프로젝트는 **비공식 auth flow** (auth.riotgames.com ssid cookie + `X-Riot-Entitlements-JWT` 교환) 를 사용한다. 라이엇이 해당 경로를 차단할 경우 서비스가 중단될 수 있으며, 이를 **수용 가능 리스크**로 간주한다. 커뮤니티 라이브러리 / 엔드포인트 변경을 상시 모니터링.
- **법적 고지**: 모든 페이지 푸터에 "VAL-Shop 은 라이엇 게임즈와 무관한 팬메이드 프로젝트입니다" 명시.

## 8. 성공 기준

MVP (주말) 수락 기준:

- **AC-1**: 로그인 성공 → 3초 이내 상점 4개 카드 (이름/가격 VP/티어 아이콘/이미지) 가 모바일 Chrome 에서 렌더된다.
- **AC-2**: 최초 로그인 후 암호화된 토큰 저장으로, 2회차 방문 시 추가 입력 없이 대시보드가 뜬다.
- **AC-3**: 상점 갱신 카운트다운이 ±1초 정확도로 실시간 갱신된다.
- **AC-4**: Chrome Lighthouse Mobile 측정 TTI ≤ 3s.
- **AC-5**: `*.vercel.app` 주소로 공개 접근 가능하고 PWA 설치 배너가 동작한다.
- **AC-6**: 1주일 운영 후 Vercel / Supabase 사용량이 무료 티어 한도 내에 머문다.

Phase 2 (여력 시) 수락 기준:

- **AC-7**: 위시리스트에 찜한 스킨이 상점 로테이션에 포함되면 1시간 이내 Web Push 가 수신된다.

측정 방법: AC-1/AC-4 Chrome DevTools + Lighthouse, AC-2 수동 재방문 테스트, AC-3 시각 확인 + setInterval 검증, AC-5 실제 URL 접속, AC-6 Vercel/Supabase 대시보드 사용량, AC-7 위시리스트 항목 등록 후 로테이션 대기 관측.
